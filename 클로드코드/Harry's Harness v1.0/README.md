# 🏗️ Harry's Harness v1.0 — Quick Start Guide

## 파일 구성

```
harness/
├── 00_ORCHESTRATOR.md      ← 전체 흐름 제어 (메인 하네스)
├── 01_PLANNER.md           ← 요청 분석 & 계획 수립
├── 02_DESIGNER.md          ← 상세 설계 문서 작성
├── 03_GENERATOR.md         ← 실제 코드 생성 (유일한 코드 작성자)
├── 04_EVALUATOR.md         ← 독립적 코드 평가 (유일한 평가자)
├── 05_SCORING_CRITERIA.md  ← 평가 기준서 (Evaluator 참조용)
└── README.md               ← 본 문서
```

---

## 실행 흐름

```
사용자 요청
    │
    ▼
┌─────────┐
│ PLANNER │ ──→ 계획 수립
└────┬────┘
     │
     ▼
┌──────────┐          ┌──────────────┐
│ DESIGNER │ ──→ ──→  │  GENERATOR   │
└──────────┘          └──────┬───────┘
                             │
                   ┌─────────▼──────────┐
                   │    EVALUATOR       │
                   │ (05_SCORING 참조)   │
                   └─────────┬──────────┘
                             │
                    ┌────────┴────────┐
                    │  PASS?          │
                    ├─ YES → 최종 출력  │
                    └─ NO  → Round +1  │
                       (Designer로 복귀)│
                    └─────────────────┘

※ 최소 3회 반복 보장 │ 합격: 총점 ≥80 & 전 카테고리 ≥60
```

---

## 핵심 규칙

| 규칙 | 설명 |
|------|------|
| 🔒 생성/평가 분리 | Generator는 평가하지 않고, Evaluator는 코드를 수정하지 않음 |
| 🔄 최소 3회 반복 | 1회 통과해도 3라운드까지 진행하여 품질 향상 |
| 📋 피드백 추적 | 매 라운드마다 이전 피드백 반영 여부를 추적 |
| 📊 기준 기반 평가 | Evaluator는 05_SCORING_CRITERIA.md에만 근거하여 평가 |
| 🚫 범위 초과 금지 | 각 에이전트는 자기 역할 범위를 넘지 않음 |

## 사용 방법

이 하네스를 사용하려면 Claude에게 다음과 같이 요청하세요:

```
00_ORCHESTRATOR.md를 읽고 이 하네스 구조에 따라 다음 코딩 작업을 수행해줘:
[당신의 코딩 요청]
```

하네스의 모든 MD 파일을 컨텍스트에 포함시키면
Claude가 각 에이전트 역할을 순차적으로 수행합니다.
