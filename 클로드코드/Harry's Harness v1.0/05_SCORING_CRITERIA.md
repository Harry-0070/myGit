# 📊 SCORING CRITERIA — 평가 기준서 v1.0

## 개요

이 문서는 Evaluator 에이전트가 코드를 평가할 때 참조하는 **독립적 평가 기준서**이다.
Generator와 완전히 분리되어 있으며, 객관적이고 일관된 평가를 보장한다.

---

## 합격 기준

```
✅ 합격: 가중 평균 총점 ≥ 80점 AND 모든 카테고리 ≥ 60점
❌ 불합격: 총점 < 80점 OR 어떤 카테고리라도 < 60점
```

---

## 평가 카테고리

### 1. 기능 정확성 (가중치: 25%)

사용자 요구사항을 정확히 충족하는가?

| 점수 범위 | 기준 |
|-----------|------|
| 90-100 | 모든 요구사항 완벽 구현. 엣지 케이스까지 처리. 추가 유용 기능 포함 |
| 70-89 | 핵심 요구사항 구현 완료. 일부 엣지 케이스 미처리 |
| 50-69 | 주요 기능 동작하나 일부 요구사항 누락 또는 버그 존재 |
| 30-49 | 기본 구조는 있으나 핵심 기능에 결함 존재 |
| 0-29 | 요구사항을 거의 충족하지 못함. 실행 불가 또는 주요 로직 오류 |

**감점 기준:**
- 핵심 기능 미구현: -20점/건
- 기능은 있으나 결과가 잘못됨: -15점/건
- 엣지 케이스 미처리: -5점/건
- 사소한 동작 차이: -2점/건

---

### 2. 코드 구조 (가중치: 20%)

코드가 잘 조직되어 있고 확장 가능한가?

| 점수 범위 | 기준 |
|-----------|------|
| 90-100 | 명확한 모듈 분리. 단일 책임 원칙 준수. 의존성이 깔끔함 |
| 70-89 | 전반적으로 잘 구조화됨. 일부 모듈 경계가 모호 |
| 50-69 | 기본 구조는 있으나 과도한 결합이나 중복 존재 |
| 30-49 | 구조가 혼란스러움. 스파게티 코드 경향 |
| 0-29 | 구조가 전혀 없음. 모든 로직이 뒤섞여 있음 |

**감점 기준:**
- God 클래스/함수 (200줄 이상의 단일 함수): -15점/건
- 순환 의존성: -10점/건
- 코드 중복 (동일 로직 3회 이상 반복): -10점/건
- 부적절한 전역 상태 사용: -5점/건
- 불필요한 복잡성 (over-engineering): -5점/건

---

### 3. 에러 처리 (가중치: 15%)

예외 상황을 적절히 처리하는가?

| 점수 범위 | 기준 |
|-----------|------|
| 90-100 | 모든 예외 경로에 적절한 처리. 사용자 친화적 에러 메시지. 복구 로직 포함 |
| 70-89 | 주요 에러 처리 완료. 일부 경로 누락 |
| 50-69 | 기본 try-catch 존재하나 구체성 부족 |
| 30-49 | 에러 처리가 거의 없거나 단순 console.log로만 처리 |
| 0-29 | 에러 발생 시 크래시하거나 무시 |

**감점 기준:**
- 빈 catch 블록 (에러 삼킴): -10점/건
- 사용자 입력 검증 누락: -8점/건
- 외부 API/파일 I/O 에러 미처리: -8점/건
- 에러 메시지에 기술적 정보만 노출: -3점/건
- 비동기 에러 미처리: -5점/건

---

### 4. 가독성 (가중치: 15%)

다른 개발자가 쉽게 이해할 수 있는가?

| 점수 범위 | 기준 |
|-----------|------|
| 90-100 | 자체 문서화된 코드. 명명이 명확. 적절한 주석. 일관된 스타일 |
| 70-89 | 전반적으로 읽기 쉬움. 일부 불명확한 변수명이나 복잡한 로직 존재 |
| 50-69 | 이해 가능하나 노력이 필요. 주석 부족 |
| 30-49 | 읽기 어려움. 약어 남용, 비일관적 스타일 |
| 0-29 | 해독 불가 수준 |

**감점 기준:**
- 단일 문자 변수명 (루프 카운터 제외): -3점/건
- 매직 넘버 사용: -3점/건
- 복잡한 로직에 주석 없음: -5점/건
- 비일관적 코딩 스타일 (들여쓰기, 명명 규칙 혼재): -5점
- 주석이 코드와 불일치: -3점/건
- 100자 초과 줄이 빈번: -2점

---

### 5. 성능 및 효율성 (가중치: 10%)

불필요한 리소스 낭비 없이 동작하는가?

| 점수 범위 | 기준 |
|-----------|------|
| 90-100 | 최적의 알고리즘 사용. 메모리 효율적. 불필요한 연산 없음 |
| 70-89 | 적절한 성능. 명백한 비효율 없음 |
| 50-69 | 동작하나 개선 여지가 있는 비효율 존재 |
| 30-49 | 성능 문제가 명백 (N+1 쿼리, 불필요한 전체 순회 등) |
| 0-29 | 심각한 성능 문제 (무한 루프 위험, 메모리 누수 등) |

**감점 기준:**
- O(n²) 이상 알고리즘 (O(n log n) 가능한 경우): -10점/건
- 불필요한 데이터 복사: -5점/건
- 메모리 누수 가능성: -10점/건
- 동기 블로킹 호출 (비동기 가능한 경우): -5점/건
- 캐싱 없이 동일 연산 반복: -3점/건

---

### 6. 테스트 커버리지 (가중치: 10%)

적절한 테스트가 포함되어 있는가?

| 점수 범위 | 기준 |
|-----------|------|
| 90-100 | 핵심 경로 + 엣지 케이스 + 에러 경로 테스트 완비. 의미 있는 어설션 |
| 70-89 | 핵심 경로 테스트 존재. 일부 엣지 케이스 포함 |
| 50-69 | 기본 테스트 존재하나 커버리지 부족 |
| 30-49 | 테스트가 매우 적거나 피상적 |
| 0-29 | 테스트 없음 또는 테스트 자체에 오류 |

**감점 기준:**
- 핵심 비즈니스 로직 테스트 없음: -15점
- 에러 경로 테스트 없음: -10점
- 테스트가 항상 통과 (의미 없는 어설션): -10점/건
- 경계값 테스트 누락: -5점
- 테스트 코드에 하드코딩된 의존성: -3점/건

---

### 7. 보안 및 안정성 (가중치: 5%)

보안 취약점이 없고 안정적으로 동작하는가?

| 점수 범위 | 기준 |
|-----------|------|
| 90-100 | 보안 모범 사례 준수. 입력 검증/이스케이프 완비 |
| 70-89 | 주요 보안 위험 없음. 기본 입력 검증 존재 |
| 50-69 | 일부 보안 고려 누락 (중요도에 따라) |
| 30-49 | 보안 취약점 존재 (SQL Injection, XSS 등) |
| 0-29 | 심각한 보안 결함. 민감 데이터 노출 |

**감점 기준:**
- SQL Injection 취약점: -20점/건
- XSS 취약점: -15점/건
- 하드코딩된 비밀번호/API 키: -15점/건
- 미검증 사용자 입력 사용: -10점/건
- 안전하지 않은 난수 생성 (보안 컨텍스트): -5점/건

---

## 총점 계산

```
총점 = Σ(카테고리 점수 × 가중치)

예시:
  기능 정확성: 85 × 0.25 = 21.25
  코드 구조:   80 × 0.20 = 16.00
  에러 처리:   70 × 0.15 = 10.50
  가독성:      90 × 0.15 = 13.50
  성능:        75 × 0.10 = 7.50
  테스트:      60 × 0.10 = 6.00
  보안:        80 × 0.05 = 4.00
  ─────────────────────────
  총점:                    78.75 → FAIL (80 미달)
```

## 평가 무결성 원칙

1. **독립성**: Evaluator는 Generator의 의도나 변명을 고려하지 않는다. 코드 자체만 평가한다.
2. **일관성**: 동일한 코드는 동일한 점수를 받아야 한다.
3. **투명성**: 모든 점수에는 구체적 근거가 있어야 한다.
4. **실행 가능성**: 모든 피드백은 Generator가 행동으로 옮길 수 있어야 한다.
5. **공정성**: 개인적 스타일 선호가 아닌, 본 문서의 기준에만 근거하여 평가한다.
